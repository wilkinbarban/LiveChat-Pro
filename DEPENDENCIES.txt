LiveChat Pro — dependencias de instalación y runtime
====================================================

Runtime
-------
El proyecto utiliza Node.js del sistema.

Requisito mínimo:
- Node.js >= 24
- npm

setup.js valida Node.js del sistema. El comando node debe existir para poder arrancar el instalador; si la versión inicial es menor que 24, intenta actualizarla en distribuciones Linux soportadas.

Comandos habituales:

  sudo npm install
  node setup.js
  sudo node server.js

Dependencias Node.js
--------------------
Definidas en package.json y bloqueadas en package-lock.json.

Producción:
- express
- socket.io
- telegraf
- sqlite / sqlite3
- dotenv
- axios
- redis
- @socket.io/redis-adapter
- pino
- helmet
- cors
- cookie-parser
- express-rate-limit
- geoip-lite
- node-cache
- sentiment

Desarrollo:
- nodemon

Dependencias del sistema para VPS
---------------------------------
Docker recomendado:
- Docker Engine
- Docker Compose plugin
- containerd
- curl
- ca-certificates
- gnupg

setup.js intenta instalar Node.js y Docker automáticamente en estas familias cuando falta:
- Debian / Ubuntu
- Fedora
- CentOS / RHEL / Rocky / AlmaLinux
- Arch Linux
- Alpine Linux

Firewall
--------
setup.js intenta abrir:
- 8080/tcp

Soporta automáticamente:
- ufw
- firewalld

Nota: si tu proveedor VPS usa security groups/firewall externo, debes abrir 8080/tcp también en el panel del proveedor.

Puertos
-------
- PORT: puerto interno de la app Node.js. Recomendado: 3000.
- HOST_PORT: puerto publicado por Docker en el VPS. Recomendado: 8080 para dejar 80/443 libres para tu sitio web público.

Para HTTPS real en 443 necesitas un proxy TLS como nginx, Caddy o Traefik y certificados.
